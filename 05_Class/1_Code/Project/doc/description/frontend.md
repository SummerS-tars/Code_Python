目前 GUI 的模糊搜索是复用后端逻辑实现的，流程如下：

- 界面只是把你在两个输入框填的文本拼成 `"起点-终点"` 传给 `MetroPathPlanner.process_user_input`，没有额外前端搜索逻辑。
- 解析层允许缺省线路：`Parser` 若未检测到分隔符“，”/“,”，就把线路记为 `None`。
- 路径查找前置处理在 `MetroPathPlanner.find_route`：
  - 若线路名缺省或按线路精确查找失败，会调用 `network.get_station_any_line(站名)` 先尝试精确站名匹配任意线路。
  - 若仍找不到，调用 `network.search_stations(keyword)` 做包含式匹配（子串匹配，不是拼音/编辑距离）。
  - 匹配结果只有 1 个则自动采用；多于 1 个则返回错误提示，列出候选并要求你补充线路或更精确站名。
- 终点与起点的处理对称。

因此，GUI 支持的“模糊”是基于站名包含关系的后端匹配，前端不做自动补全或下拉选择；若有重名/多匹配会提示你进一步指定线路。
