### 第一部分：当前完成度与新目标评估

#### 1. 现有完成度回顾

* **核心功能 (Core)**: 路径规划算法（BFS/Dijkstra）已重构完成，支持“最少换乘”策略，逻辑稳健。
* **数据层 (Data)**: `DataLoader` 解析 CSV 正常，`fetch_data.py` 脚本验证有效，但目前是独立的。
* **界面 (UI)**: 基础 GUI 已跑通，但交互形式（纯文本输入）和展示形式（纯文本输出）较为初级。

#### 2. 新需求分析与优化方向

* **集成 `fetch_data**`: 目前更新数据需要手动运行脚本。优化方向是将爬虫逻辑封装为服务类（Service），在 GUI 中一键调用，实现“在线更新 -> 自动重载 -> 立即生效”。
* **数据可视化 (Data Viewer)**: 用户目前是个“瞎子”，不知道系统里到底有哪些站。我们需要一个**层级树状视图 (Tree View)**，清晰展示 `线路 -> 站点` 结构，作为系统的“字典”。
* **保留的优化**: **智能输入 (Autocomplete)** 依然非常有必要，它是连接“用户模糊记忆”与“系统精确数据”的桥梁，必须保留。
