## 第11讲作业：开发问卷数据收集工具
### 一、 作业目标

本作业旨在综合运用本周所学的核心技能，让你能够开发一个完整、健壮且实用的命令行应用程序。完成此作业后，你应该能够：
- 处理结构化数据：熟练使用 json 模块读取和写入包含复杂数据结构（字典列表）的文件。
- 实现数据持久化：让你的程序拥有“记忆”，能够保存多次运行产生的数据。
- 编写健壮的程序：使用 try...except 结构处理常见的文件和用户输入异常，防止程序意外崩溃。
- 巩固模块化思想：将程序的不同功能（主逻辑、文件处理）分离到不同模块中，编写清晰、可维护的代码。

### 二、 核心功能要求

你需要开发一个程序，该程序可以循环执行“收集-保存”问卷数据的流程。

#### 1. 问卷设计
程序需要向用户提出以下三个问题：
- 问题一：“您的姓名是？” (类型: 字符串)
- 问题二：“您的年龄是？” (类型: 整数)
- 问题三：“您最喜欢的编程语言是？(可选: Python, Java, C++, JavaScript, Other)” (类型: 字符串)

#### 2. 数据收集与存储
- 内存结构: 每份完整的问卷回答在程序内存中应存储为一个字典。例如：{'name': '张三', 'age': 25, 'language': 'Python'}。
- 文件格式: 所有收集到的问卷数据必须以 JSON 格式 保存到一个名为 surveys.json 的文件中。
- 存储结构: surveys.json 文件的顶级结构应该是一个列表 []，其中每个元素都是一份问卷的字典。
- 追加逻辑: 程序不能覆盖已有的问卷数据。每次成功收集一份新问卷后，程序需要：
    - 首先，从 surveys.json 文件中读取已有的问卷列表。
    - 然后，将新的问卷字典追加到这个列表中。
    - 最后，将更新后的整个列表写回到 surveys.json 文件中。

#### 3. 循环与退出
每当一份问卷被成功保存后，程序必须询问用户：“是否要继续进行下一份问卷调查？(yes/no)”。
- 如果用户输入 yes (不区分大小写)，则开始新一轮的问卷提问。
- 如果用户输入 no (不区分大小写)，则程序打印“感谢您的参与，程序已退出。”并正常终止。

### 三、 技术与代码结构要求
#### 1. 模块化
你的代码必须至少包含两个模块：
- main.py (或 survey.py): 包含程序的主逻辑，如定义问卷问题、控制问卷流程、调用其他模块函数等。
- storage.py: 包含所有与文件操作相关的代码。应至少提供两个函数：
    - load_data(filepath): 负责从JSON文件加载数据。
    - save_data(filepath, data): 负责将数据写入JSON文件。

#### 2. 健壮性 (异常处理)
你的程序必须能够优雅地处理以下异常情况，而不是直接崩溃：
- FileNotFoundError: 当程序首次启动且 surveys.json 文件还不存在时，load_data 函数不应报错，而应返回一个空的列表 [] 作为初始数据。
- ValueError: 当询问用户年龄时，如果用户输入了非数字内容（例如 "abc"），程序应捕获此异常，向用户显示错误提示（如“年龄必须是数字，请重新输入！”），并重新要求用户输入年龄，直到输入正确为止。
- json.JSONDecodeError (挑战任务): 如果 surveys.json 文件因某些原因被损坏（例如，内容不是合法的JSON格式），load_data 函数应能捕获此异常，并返回一个空列表，同时给用户一个警告提示。
